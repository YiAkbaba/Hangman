# Supabase Datenbank-Setup-Anleitung (Highscores)

Diese Anleitung erklärt Schritt für Schritt, wie Sie **Supabase** einrichten, um die Highscores für das Cyber-Terminal Hangman-Spiel zu speichern und live auf dem Startbildschirm anzuzeigen.

---

## 1. Supabase-Konto & Projekt erstellen

1. Registrieren Sie sich kostenlos auf [supabase.com](https://supabase.com/).
2. Klicken Sie im Dashboard auf **New Project** (Neues Projekt).
3. Wählen Sie eine Organisation aus, geben Sie dem Projekt einen Namen (z.B. `Hangman-Term`) und setzen Sie ein sicheres Datenbank-Passwort.
4. Wählen Sie eine Region in Ihrer Nähe aus und klicken Sie auf **Create new project**. Das Projekt wird nun in ca. 1-2 Minuten bereitgestellt.

---

## 2. SQL-Tabelle "highscores" anlegen

Um Scores zu speichern, benötigen wir eine Tabelle namens `highscores`. Supabase bietet einen integrierten SQL-Editor, mit dem Sie die Tabelle in Sekundenschnelle anlegen können:

1. Navigieren Sie in der linken Seitenleiste zum **SQL Editor** (Symbol mit der SQL-Schriftrolle).
2. Klicken Sie auf **New Query** (Neue Abfrage).
3. Kopieren Sie den folgenden SQL-Code und fügen Sie ihn in den Editor ein:

```sql
-- Erstellt die Tabelle 'highscores' für die Agenten-Bestenliste
CREATE TABLE public.highscores (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    username text NOT NULL,
    score bigint NOT NULL,
    created_at timestamp with time zone DEFAULT now() NOT NULL
);

-- Aktiviert Zeilensicherheitsrichtlinien (Row Level Security - RLS)
ALTER TABLE public.highscores ENABLE ROW LEVEL SECURITY;

-- Richtlinie 1: Erlaubt jedem (auch anonymen Besuchern), Einträge zu lesen (z.B. für Top 5)
CREATE POLICY "Erlaube Lesen fuer alle" 
ON public.highscores FOR SELECT 
TO anon 
USING (true);

-- Richtlinie 2: Erlaubt anonymen Besuchern das Einfügen von neuen Highscores
CREATE POLICY "Erlaube Einfuegen fuer alle" 
ON public.highscores FOR INSERT 
TO anon 
WITH CHECK (true);
```

4. Klicken Sie rechts unten auf **Run** (Ausführen).
5. Nach erfolgreicher Ausführung steht die Tabelle bereit! Sie können in der linken Leiste unter **Table Editor** überprüfen, ob die Spalten `id`, `username`, `score` und `created_at` angelegt wurden.

## 3. API-Schlüssel in "script.js" eintragen

Um Ihr Web-Terminal mit der Datenbank zu verbinden, müssen Sie die anonymen API-Schlüssel in Ihre `script.js` Datei kopieren:

1. Navigieren Sie in Supabase zu den **Project Settings** (Zahnrad-Symbol ganz unten links) und wählen Sie den Reiter **API**.
2. Kopieren Sie die **Project URL** (z.B. `https://xyzabc.supabase.co`).
3. Kopieren Sie den **anon public API Key** (ein sehr langer kryptischer String).
4. Öffnen Sie die Datei [script.js](file:///c:/Users/avess/Downloads/Github/Hangman/script.js) in Ihrem Editor.
5. Suchen Sie ganz oben die Zeilen 11-12 und ersetzen Sie die Platzhalter:

```javascript
// ==========================================
// SUPABASE CONFIGURATION & CREDENTIALS
// ==========================================
const SUPABASE_URL = "HIER_IHRE_SUPABASE_URL_EINTRAGEN";
const SUPABASE_ANON_KEY = "HIER_IHREN_SUPABASE_ANON_KEY_EINTRAGEN";
```

## 4. Offline-Sicherheitsnetz (LocalStorage Fallback)

> [!TIP]
> **Sofort spielbar ohne Setup!**
> Die App verfügt über ein integriertes Offline-Sicherheitsnetz. Wenn Sie das Supabase-Setup überspringen oder die Platzhalter `"YOUR_SUPABASE_URL"` und `"YOUR_SUPABASE_ANON_KEY"` unverändert lassen, schaltet das Skript **automatisch und ohne Fehlermeldung** auf den **LocalStorage-Modus** um.
>
> Highscores werden dann lokal im Browser des Spielers gespeichert. Sobald Sie gültige Supabase-Schlüssel eintragen, verbindet sich das Spiel beim nächsten Neuladen direkt mit Ihrer globalen Cloud-Bestenliste!
